## Task 3: Room Join Page — Default to Login, Guest Cookie Persistence

**Goal:** 
- Default to email/password login (not guest)
- "ورود مهمان" button below login
- Persist guest session in cookies so returning guests don't need to re-enter name
- Admin can join room directly from /admin/rooms

**Files to modify:**
- `web/src/routes/room/[slug]/+page.svelte` — Reorder UI: login first, guest second
- `web/src/routes/(app)/admin/rooms/+page.svelte` — Add "ورود" button per room
- `web/src/lib/stores.ts` — Add cookie-based guest session persistence

### 3a: Room join page restructure

The page should show:
1. Room info (name, description, color)
2. **Email input** (default, always visible)
3. **Password input** (default, always visible)
4. **"ورود" button** (login with credentials)
5. Divider or spacer
6. **"ورود مهمان" button** (if guest_login_enabled)
   - If returning guest (cookie exists): auto-fill name from cookie, skip name input
   - If new guest: show name input

- [ ] Rewrite `room/[slug]/+page.svelte` with new layout

### 3b: Cookie-based guest persistence

When a guest joins a room:
1. After successful `room-guest-login`, save `{ roomSlug, displayName, tokens }` in a cookie (e.g., `iroom_guest_{slug}`)
2. On page load, check for this cookie
3. If cookie exists and tokens are valid, auto-redirect to `/room/{slug}/live`
4. If cookie exists but tokens expired, show name pre-filled (just click "ورود مهمان")

Cookie approach:
- Use `document.cookie` with `path=/`, `max-age=2592000` (30 days), `SameSite=Lax`
- Store JSON: `{ displayName, accessToken, refreshToken }`
- For security: encrypt or at least encode the tokens (base64)

- [ ] Add cookie helper functions in `stores.ts` or a new `$lib/cookie.ts`
- [ ] On guest login success: save cookie
- [ ] On page load: check cookie, pre-fill name or auto-login
- [ ] On logout: clear cookie

### 3c: Admin join button

Add a "ورود" button in admin/rooms that opens `/room/{slug}` in a new tab (admin joins as themselves).

- [ ] Add join button to admin/rooms card
