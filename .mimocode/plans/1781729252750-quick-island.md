## Root Cause Analysis

### PION WEBRTC (Critical — class won't load)

**Issue 1 [CRITICAL]: Room never created before adding participants**
- File: `internal/webrtc/signaling.go` line 96
- `HandleOffer()` calls `AddParticipant()` but never calls `GetOrCreateRoom()` first
- First participant always gets `ErrRoomNotFound` → HTTP 409 → connection fails silently

**Issue 2 [HIGH]: Error format mismatch**
- Backend returns `echo.NewHTTPError()` → `{"message": "..."}`
- Frontend expects `{"success": false, "error": "..."}` 
- Errors silently swallowed

**Issue 3 [HIGH]: ICE candidate missing required fields**
- Frontend sends `candidate` string only
- Backend needs `sdpMid` and `sdpMLineIndex` for Pion to match candidates

**Issue 4 [MEDIUM]: broadcastToRoom writes JSON to RTP tracks**
- Control messages (join/leave) written to video RTP streams → corrupts video
- Should use DataChannel instead

**Issue 5 [LOW]: GetJoinInfo doesn't validate session status**
- Backend should reject if session != "live"

### PWA ServiceWorker (Minor)

**Issue 6: `/favicon.svg` missing from static/**
- `sw.js` tries to cache it in `addAll` → entire install fails

**Issue 7: Deprecated meta tag**
- `apple-mobile-web-app-capable` → use `display: standalone` in manifest
