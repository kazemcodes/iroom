# IRoom - Open Source Online Classroom Platform

## Project Overview
An open-source online classroom platform (like Skyroom/BlueJeans) designed for Iranian users, fully Persian RTL, self-hostable on any VPS (1 vCPU / 1GB RAM), supporting 100+ concurrent users.

---

## Recommended Technology Stack

### Backend
| Component | Choice | Why |
|-----------|--------|-----|
| Language | **Go (Golang)** | Ultra low memory (~50MB), single binary |
| HTTP Framework | **Echo** | Lightweight, fast, great middleware |
| WebRTC SFU | **LiveKit Server (sidecar)** | Battle-tested Go SFU, handles 100+ users, ~120MB RAM |
| Database | **SQLite (WAL mode)** | Zero-config, concurrent reads, ~5MB RAM |
| Query Builder | **sqlx** | Lightweight, no heavy ORM |
| Auth | **JWT (custom)** | Minimal, full control |
| WebSocket | **gorilla/websocket** | Used by LiveKit itself |
| Logging | **slog (stdlib)** | Zero dependencies |

### Frontend
| Component | Choice | Why |
|-----------|--------|-----|
| Framework | **SvelteKit** | Lightweight, fast, familiar to user |
| CSS | **TailwindCSS + RTL plugin** | RTL-first, utility classes |
| WebRTC Client | **livekit-client** | Official JS SDK |
| Whiteboard | **Fabric.js** | 200KB, mature, great Persian text support |
| Icons | **Lucide Icons** | Clean, RTL-friendly |
| Calendar | **date-fns-jalali** | Persian Jalali calendar |

### Infrastructure
| Component | Choice | Why |
|-----------|--------|-----|
| Reverse Proxy | **Caddy** | Auto-HTTPS, lighter than nginx |
| Deployment | **Docker Compose** | Single `docker-compose up` |
| TURN Server | **coturn (optional)** | Only for strict NAT environments |

## Architecture

### Memory Budget (1GB VPS)
```
LiveKit Server:     ~100-120 MB
Go Backend:         ~30-50 MB
SQLite:             ~5-10 MB
Caddy:              ~10-15 MB
OS Overhead:        ~100-150 MB
Buffer:             ~50-100 MB
────────────────────────────
Total:              ~300-460 MB (fits in 1GB)
```

### Deployment Topology
```
┌──────────────────────────────────────────────┐
│              VPS (1 vCPU / 1GB RAM)          │
│                                               │
│  Caddy (:443) → Go Backend (:8080)           │
│                 ↓                             │
│              LiveKit SFU (:7880/7881)        │
│              SQLite (WAL mode)               │
└──────────────────────────────────────────────┘
```

## Core Features & Implementation

### 1. WebRTC Video/Audio Conferencing (LiveKit)
- Teacher creates room, generates join tokens for students
- Students get audio-only publish by default (bandwidth savings)
- Teacher can grant video publish to specific students
- Screen sharing via LiveKit screen track
- Simulcast enabled for bandwidth adaptation

### 2. Whiteboard (Fabric.js + LiveKit DataChannels)
- Fabric.js canvas with tools: pen, shapes, text, eraser
- Sync via LiveKit DataChannels (no extra WebSocket server needed)
- Protocol: `{ op: 'add'|'update'|'delete'|'clear', data: serializedFabricObject }`
- Full state sync when new participant joins

### 3. Chat (WebSocket)
- Real-time text chat per session
- REST fallback for message history
- Virtual scrolling for 100+ messages

### 4. Client-Side Recording
- MediaRecorder API (browser-native, no server CPU needed)
- WebM/VP9 format, uploaded on session end
- Playable via HTML5 video player

### 5. Class Scheduling (Jalali Calendar)
- date-fns-jalali for Persian dates
- Recurring sessions support
- Session status: scheduled → live → ended

### 6. User Management
- Roles: admin, teacher, student
- JWT auth with refresh tokens
- Teacher creates classes, enrolls students

### 7. File Sharing
- Multipart upload to server
- Download links per session
- Stored in uploads/ directory

## Database Schema (6 tables)

- **users** - id, email, password_hash, display_name, role, phone, is_active
- **classes** - id, teacher_id, name, description, color, max_students
- **class_students** - class_id, student_id (many-to-many)
- **sessions** - id, class_id, title, scheduled_at, duration, status, livekit_room, recording_url
- **messages** - id, session_id, user_id, content, type
- **files** - id, session_id, uploaded_by, filename, filepath, filesize

## Key API Endpoints

### Auth & Users
```
POST   /api/v1/auth/register|login|refresh
GET    /api/v1/auth/me
PUT    /api/v1/auth/me                    # Update profile
```

### Classes & Sessions
```
GET|POST    /api/v1/classes
POST        /api/v1/classes/:id/enroll
GET|POST    /api/v1/sessions
POST        /api/v1/sessions/:id/start|end|join
```

### Communication
```
GET|POST    /api/v1/sessions/:id/messages
WebSocket   /ws/sessions/:id
POST        /api/v1/files/upload
GET         /api/v1/files/:id
```

### LiveKit
```
POST   /api/v1/livekit/webhook
```

### Admin Panel (admin role required)
```
GET    /api/v1/admin/dashboard/stats
GET    /api/v1/admin/users               # List all users (paginated, searchable)
POST   /api/v1/admin/users               # Create user
PUT    /api/v1/admin/users/:id           # Update user
DELETE /api/v1/admin/users/:id           # Deactivate user
GET    /api/v1/admin/classes             # List all classes
POST   /api/v1/admin/classes             # Create class
PUT    /api/v1/admin/classes/:id         # Update class
DELETE /api/v1/admin/classes/:id         # Delete class
GET    /api/v1/admin/sessions            # List all sessions
GET    /api/v1/admin/sessions/:id        # Session details
GET    /api/v1/admin/recordings          # List all recordings
DELETE /api/v1/admin/recordings/:id      # Delete recording
GET    /api/v1/admin/logs                # Activity log
PUT    /api/v1/admin/settings            # System settings
```

### Remote Management API (API key auth)
```
POST   /api/v1/external/users            # Create user from LMS/CMS
POST   /api/v1/external/classes          # Create class
POST   /api/v1/external/sessions         # Schedule session
GET    /api/v1/external/status           # System health check
GET    /api/v1/external/stats            # Usage statistics
POST   /api/v1/external/webhooks         # Register webhook URL
```

## Deployment

### Quick Start (Docker)
```bash
git clone https://github.com/parsiclass/parsiclass.git
cd parsiclass
cp .env.example .env
# Edit .env with your secrets
docker-compose up -d
```

### Bare Metal (systemd)
```bash
# Build Go binary
go build -o server ./cmd/server

# Run with systemd
sudo cp deploy/classroom.service /etc/systemd/system/
sudo systemctl enable --now classroom
```
