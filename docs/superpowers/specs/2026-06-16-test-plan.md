# IRoom — Comprehensive Test Plan

**Date:** 2026-06-16  
**Type:** Test Plan & Strategy  
**Scope:** Full-stack testing for all IRoom features

---

## 1. Test Strategy Overview

### 1.1 Testing Levels

| Level | Scope | Tools | Who |
|-------|-------|-------|-----|
| **Unit** | Individual functions, utilities, components | Vitest, Svelte Testing Library | Developers |
| **Integration** | API handlers, database queries, component interactions | Vitest, Supertest | Developers |
| **E2E** | Full user flows, cross-browser | Playwright | QA + Developers |
| **Performance** | Load, stress, memory | k6, Lighthouse | QA |
| **Security** | Auth, authorization, input validation | Manual + OWASP ZAP | QA + Security |
| **Accessibility** | WCAG 2.1 AA compliance | axe-core, Manual | QA |
| **Visual** | UI consistency, RTL layout | Chromatic, Manual | QA + Design |

### 1.2 Test Environments

| Environment | Purpose | Data |
|-------------|---------|------|
| **Local** | Development testing | Seed data |
| **CI** | Automated test runs on PR | Fresh seed per run |
| **Staging** | Pre-production validation | Anonymized production-like |
| **Production** | Smoke tests, monitoring | Real data |
